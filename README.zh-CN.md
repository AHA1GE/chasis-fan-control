# 机箱风扇控制器

基于 **CH32V003** RISC‑V 微控制器的双通道机箱风扇控制器。固件可同时驱动两路 4 针 PWM 风扇，提供按键速度选择、WS2812 电量指示以及转速（tach）回读——所有功能由 1S–3S LiPo 电池供电，并具备自动电芯数检测。

## 功能

- **双通道风扇 PWM 控制** — 两路风扇运行相同占空比，由 CH32V003 的硬件定时器产生 PWM。
- **按键速度选择** — 单击循环切换 0% → 25% → 50% → 75%；双击跳到 100%；长按立即回到 0%。
- **转速回读（Tach）** — 从风扇的开漏转速信号读取 RPM，并把转速映射为电量指示 LED 的线性热图颜色（低速蓝 → 中速绿 → 高速红）。
- **电量指示** — 4 个 WS2812C LED 以“计数”方式显示电量：1 灯 = 5–25%，2 = 25–50%，3 = 50–75%，4 = 75–100%。所有点亮的 LED 共享相同的转速热图颜色；当电量临界（<10%）时，顶部 LED 以 2 Hz 闪烁警告。
- **自动电芯数检测** — 上电时采样电压分压器判断 1S（3.0–4.2 V）、2S（6.0–8.4 V）或 3S（9.0–12.6 V）锂电池组。
- **物理主开关** — 通过 N 沟道 MOSFET 将电池地完全断开；关机时实现零静态电流。
- **升降压稳压器** — 使用 TPS63070 提供稳定的 5 V 输出（2–16 V 输入范围），可覆盖所有支持电池的放电范围。
- **AIO 接针（H1）** — 5 针台座将 SWD（WCH‑LinkE / J‑Link）与保留的 WS2812 扩展引脚合并（第五脚目前未在固件中使用）。
- **灵活的风扇接入** — 主 CN1 插座支持两只风扇；可选 CN4 接针支持单风扇接入。固件可自适应：双风扇时合并转速读取，单风扇时退回到单一路读取。

## 使用说明

### 上电

1. 连接风扇：将两只风扇插入 CN1，或将单只风扇插入 CN4（两者为替代接口，按需选择）。
2. 连接 1S–3S LiPo 电池（电池电源接入 CN1 的 7–8 脚或焊盘）。
3. 将滑动开关（SW2）拨到 **ON** —— 升降压开始工作，MCU 启动，4 个 WS2812 LED 播放上电动画。 
4. MCU 采样电压分压器并判定电芯数，对应数量的 LED 点亮显示电量。

### 风扇速度控制

| 操作 | 动作 |
|------|------|
| **单击** | 循环占空比：**0% → 25% → 50% → 75%**（循环回到 0%） |
| **双击** | 跳到 **100%** 占空比 |
| **长按** | 立即回到 **0%** 占空比 |

上电后风扇初始为 0%（静止或最低转速，取决于风扇）。当前占空比由 WS2812 LED 显示。

### 读取风扇速度

所有点亮的电量 LED 共享同一转速热图颜色——电量只决定点亮的 LED 数量，而不影响颜色：

| 颜色 | 含义 |
|------|------|
| 🔵 蓝 | 最低 RPM（≤ 2000） |
| 🟢 绿 | 中等 RPM（约 11000） |
| 🔴 红 | 最高 RPM（≥ 20000） |

颜色在蓝 → 绿 → 红 之间线性过渡，随 RPM 连续变化。

### 电量指示

| 点亮 LED 数 | 电量 |
|-----------|------|
| 1 灯 | 5% – 25% |
| 2 灯 | 25% – 50% |
| 3 灯 | 50% – 75% |
| 4 灯 | 75% – 100% |

当电量处于临界（<10%）时，最顶部的 LED 以 2 Hz 闪烁。

### 关机

将滑动开关拨到 **OFF** —— MOSFET 断开电池地，整板实现零静态电流。

### 实现说明

- **以 Arduino 风格为主** — 高层逻辑使用 `pinMode`、`digitalRead`、`analogRead`、`millis`、`delayMicroseconds`。寄存器级初始化被封装在三个初始化助手函数中（`pwm_begin`、`ws2812_begin`、`tach_begin`），仅在 `setup()` 中调用。
- **不禁用中断** — WS2812 通过 3 MHz 硬件 SPI 驱动。48 字节的传输（4 个 LED × 3 色 × 每色 4 个 SPI 字节）通过 `SPI1->DATAR` 串行发送，无需禁用中断。tach 的 ISR 可以正常触发。
- **RISC‑V 中断锁定** — tach 计数的读取/清零使用 `mstatus.MIE` CSR 操作（`irq_lock` / `irq_restore`）以保证原子性，而不是 Arduino 的 `noInterrupts()`/`interrupts()`，因为部分 CH32V003 内核不支持那些 API。
- **按键消抖** — 在 `loop()` 中以约 1 ms 的速率轮询，30 次采样计数消抖。通过下/上沿检测实现单击/双击/长按的识别，时序由间隔超时决定。
- **转速（Tach）** — 在 PD2 与 PA1 上 `attachInterrupt`（EXTI2/EXTI1）增加原子脉冲计数。主循环每 2 秒快照并清零计数器，计算 RPM = 脉冲数 × 15。
- **启动序列** — 2 s 彩虹动画 → 采样电池电压并判定电芯数（4 次采样取平均）→ 全速（100%）运行 500 ms → 记录 1 s 的 tach 脉冲 → 进入 RUNNING 或 ERROR 状态。

### 显示行为

| 显示 | 含义 |
|------|------|
| 4× 彩虹旋转（2 s） | 上电启动动画 |
| 4 灯快速红色闪烁（5 Hz） | 错误：电池电压异常或未检测到风扇 |
| 1–4 灯点亮，颜色 = 蓝 → 绿 → 红 | 电量：灯数表示 SoC；颜色表示风扇 RPM 热图（所有点亮 LED 颜色相同） |
| 顶部 LED 以 2 Hz 闪烁 | 电量临界（<10%），闪烁显示当前 RPM 颜色 |

## 测试

固件未在硬件上测试。实物到手后按以下顺序验证：

### 1. PWM（示波器）
探针 PD5 与 PD6。确认 25 kHz，空载为 0% 占空。通过按键切换占空，确认 CCR 值对应 `{0, 480, 960, 1440, 1919}` 分别代表 `{0%, 25%, 50%, 75%, 100%}`。

### 2. WS2812（逻辑分析仪）
探测 PC6（SPI1_MOSI）。确认 3 MHz 时钟，SPI 字节流正确（`0x88`/`0x8E`/`0xE8`/`0xEE` 模式），每帧 48 字节，复位间隔 ≥ 50 µs。目视确认彩虹动画、RPM 热图颜色一致、以及电量 LED 数与 SoC 匹配。

### 3. 电池 ADC（可变直流电源）
对 BAT+ 施加已知电压并读取 `analogRead(PA2)`。确认：
- 1S 检测：ADC 读数 200–340
- 2S 检测：ADC 读数 370–570
- 3S 检测：ADC 读数 > 570
- 超出范围 → 进入 ERROR 状态并快速红色闪烁

### 4. Tach（信号发生器或实际风扇）
对 PD2 与 PA1 施加 10–500 Hz 方波。确认脉冲计数及 RPM 计算（RPM = 脉冲数 × 15）正确。验证单风扇与双风扇模式在上电时能被正确识别。

### 5. 按键（串口调试或 LED 反馈）
测试每种手势：
- 单击 → 占空循环 0 → 25 → 50 → 75 → 0%
- 双击 → 直接跳到 100%
- 长按（>1 s） → 立即回到 0%

### 6. 端到端
整体验证：上电 → 彩虹动画 → 电量指示（灯数 = SoC，颜色 = RPM 热图）→ 按键控制风扇速度 → 转速颜色随 RPM 跟随 → 低电量触发顶部 LED 2 Hz 闪烁 → 关机。

### 边界情况
- 上电无电池 → ERROR
- 上电有电池但无风扇 → ERROR
- 运行中热插拔风扇 → 所有点亮 LED 颜色降为蓝（0 RPM）
- 上电时 ADC 值处于阈值边界（1S/2S） → 正确判定电芯数

## 构建与烧录

1. 安装 CH32V003 Arduino 开发板包（例如 openwch 的 arduino_core_ch32 或 ch32v003fun）。
2. 在 Arduino IDE（或支持的 PlatformIO）中打开 `fanControl/fanControl.ino`。
3. 选择开发板：**CH32V003F4P6**。
4. 使用 WCH-LinkE 通过 H1 连接：`GND – 5V – SWD(PD1) – NRST`。
5. 编译并上传。

> **寄存器名兼容性：** 固件使用 WCH SPL 的寄存器命名（例如 `TIM1->ATRLR`、`SPI1->CTLR1`、`AFIO->PCFR1`）。如果你的 Arduino core 使用不同名称（例如 `TIM1->ARR`、`SPI1->CR1`），请调整三个初始化助手函数（固件中的 4、5、8 节）。`NVIC_EnableIRQ` 可能需要替换为 `PFIC_EnableIRQ`，这取决于 core 版本。
